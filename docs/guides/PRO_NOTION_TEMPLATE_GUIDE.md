# 🐹 하루치 Pro 2.0 노션 템플릿 구현 가이드

> 기존 기본 버전 임베드·DB를 Pro 2.0 템플릿에 통합하는 방법과 레이아웃·UX/UI 가이드라인

---

## 📋 개요

| 구분 | 기본 버전 | Pro 2.0 버전 |
|------|-----------|--------------|
| **임베드 URL** | `haruchi.vercel.app/?notion=1` | `haruchipro.vercel.app/?notion=1` |
| **XP 연동** | 수동 (밥/물/쓰담 클릭) | 자동 (노션 체크 → XP 반영) |
| **DB 구조** | 동일 | 동일 (기존 DB 그대로 사용 가능) |

---

## 1. 기존 기본 버전 → Pro 2.0 마이그레이션

### 1-1. 임베드 URL만 변경

기존 "Yohan Stodio" 등 기본 버전 페이지에 하루치 임베드가 있다면:

| 변경 전 | 변경 후 |
|---------|---------|
| `https://haruchi.vercel.app/?notion=1` | `https://haruchipro.vercel.app/?notion=1` |

**방법**: 노션에서 임베드 블록 클릭 → URL 수정

### 1-2. DB는 그대로 사용

기존 DB들(할일, 루틴, 운동, 독서, 책형, SNS, XP, 하루치)은 **Pro 2.0에서도 동일하게 사용**합니다.

- notion-sync가 같은 DB 구조를 읽어 XP 로그 생성
- Vercel 환경변수에 DB ID만 설정되어 있으면 됨

## 2. Pro 2.0 템플릿 권장 구조

### 2-1. 전체 레이아웃 (상→하)

```
[하루치 Pro 2.0] 더 똑똑해진 나만의 갓생 메이트 ✨
│
├─ 🎯 오늘의 목표 (Callout)
│   └─ 하나만 정하면 돼요. 오늘 꼭 해내고 싶은 한 가지는?
│
├─ 🐹 하루치 (Embed) ← haruchipro.vercel.app/?notion=1
│   └─ 높이 450px 이상 권장
│
├─ ✅ 오늘 할 일 (To-do 또는 Linked DB: 할일 DB)
│   └─ 작은 것부터 하나씩. 체크할 때마다 하루치가 자라요.
│
├─ 📝 하루 일기 (Toggle)
│   └─ 세 줄이면 충분해요. 오늘 하루, 어땠나요?
│
├─ 🌱 씨앗 농장 (Callout)
│   └─ 매일 한 가지라도 체크하면, 씨앗이 해바라기가 돼요.
│
└─ [선택] DB 링크 섹션
    ├─ 할일 DB
    ├─ 루틴 DB
    ├─ 운동 DB
    ├─ 독서 DB
    ├─ SNS DB
    └─ XP DB / 하루치 DB
```

### 2-2. 기존 DB 통합 방식

| 방식 | 설명 | Pro 2.0 추천 |
|------|------|-------------|
| **Inline DB** | 페이지 안에 DB 전체 삽입 | ❌ 공간 많이 차지 |
| **Linked DB** | "다른 DB 연결"로 뷰만 삽입 | ✅ 할일·루틴 등 |
| **DB 링크** | DB 페이지로 이동하는 링크만 | ✅ 상세 관리용 |

**권장**: "오늘 할 일"은 **Linked DB(할일 DB)** 로 삽입 → 체크 시 notion-sync가 XP 생성 → Pro 임베드가 30초마다 XP 반영

## 3. UX/UI 디자인 가이드라인

### 3-1. 화이트 스킨 스타일 (Pro 2.0 기본)

- **배경**: 흰색 또는 연한 회색 (`#fafafa`)
- **텍스트**: 진한 회색 (`#37352f`) 또는 검정
- **강조**: 민트/그린 계열 (`#0f380f`, `#8bac0f`) — 하루치 브랜드 컬러

### 3-2. 섹션 간격

| 요소 | 권장 |
|------|------|
| 제목 ↔ 첫 섹션 | 24px |
| 섹션 간격 | 16~20px |
| 임베드 주변 | 12px 패딩 |

### 3-3. 모바일 고려

- 노션 앱에서도 임베드 정상 동작
- 1칼럼 레이아웃 시: 하루치 임베드 → 할 일 → 일기 순으로 세로 배치
- 임베드 높이 450px 이상 유지 (모바일에서 잘림 방지)

### 3-4. 접근성

- Callout에 이모지 + 텍스트 병기
- 링크는 "DB 이름 + 아이콘" 형태로 명확히 표시
