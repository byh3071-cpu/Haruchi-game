## 👤 고객 점검·운영 관리

### 현재 가능한 확인 사항
| 항목 | 설명 | 방법 |
|------|------|------|
| 접속 수 | 방문자 수 파악 | Google Analytics 등 추가 시 |
| 오류 발생 | 게임 오작동 | 사용자 피드백 or Sentry 등 |
| 브라우저/기기 | 호환성 문제 | 피드백 수집 |

### 피드백 수집 (추가 권장)
- **이메일/폼**: 게임 내 "문의하기" 링크 → 피드백 폼(Google Forms 등)
- **SNS**: 공유 링크와 함께 DM/댓글로 의견 받기
- **버그 제보**: 이슈 트래커(GitHub Issues) 또는 노션 등으로 모음

### 향후 도입 시 고려
- **Google Analytics (GA4)**: 방문 수, 체류 시간, 기기/브라우저 통계
- **Sentry**: 자바스크립트 오류 자동 수집·알림
- **Vercel Analytics**: Vercel 유료 플랜에서 방문 통계

## 📊 데이터 분석

### 현재 구조의 한계
- **localStorage** 사용 → 서버에 데이터가 없음
- 사용자별 레벨·플레이 데이터는 **각자 기기에만** 저장
- 관리자 입장에서는 **집계·분석 불가**

### 데이터 분석을 위한 도입 옵션
| 도구 | 용도 | 비용 |
|------|------|------|
| Google Analytics 4 | 방문자 수, 페이지뷰, 유입 경로 | 무료 |
| Vercel Analytics | 방문 수, 국가별 통계 | 유료 |
| Mixpanel / Amplitude | 행동 분석, 전환 퍼널 | 무료 티어 있음 |
| 백엔드 API + DB | 레벨·EXP 서버 저장 | 별도 개발 필요 |

### GA4 기본 지표 예시
- 일/주/월 방문자 수
- 접속 기기 (PC/모바일)
- 유입 경로 (직접/카톡/검색 등)
- 평균 체류 시간

## Vercel로 배포 (링크 공유용)

1. **GitHub에 푸시**
   ```bash
   git add .
   git commit -m "공유용 버전"
   git push origin main
   ```

2. **Vercel 연결**
   - [vercel.com](https://vercel.com) 로그인
   - "Add New Project" → GitHub 저장소 선택
   - "Import" 클릭 (설정은 그대로 사용)
   - Deploy 완료 후 URL 획득 (예: `haruchi-game.vercel.app`)

3. **링크 공유**
   - 배포된 URL을 카톡, SNS 등에 공유
   - 사용자가 링크 클릭 시 바로 게임 플레이 가능
